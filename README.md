# ER Wait Time Analysis

## Project Overview
This project analyzes emergency room (ER) wait times across hospitals to identify bottlenecks, staffing impacts, and patient satisfaction trends.
The goal is to uncover operational insights that hospital administrators can use to reduce patient wait times.

---

## Dataset
Simulated ER operational dataset containing:

- Hospital information
- Staffing levels
- Patient urgency levels
- Wait time stages
- Patient satisfaction
- Visit time patterns

---

## Tools Used

Python (Pandas, Matplotlib)  
SQL (PostgreSQL)  
Jupyter Notebook  

---

## Project Phases

### Phase 1 – Data Cleaning
- Standardized column names
- Created weekend flags
- Engineered wait time stage percentages
- Created wait time buckets

### Phase 2 – Exploratory Data Analysis
Key insights discovered:

- Evening hours have the longest wait times
- Moderate urgency patients experience longer waits
- Hospitals with higher nurse ratios have shorter waits

### Phase 3 – SQL Business Analysis
SQL queries were used to analyze:

- Peak wait periods
- Hospital performance comparison
- Weekend vs weekday operations

### Phase 4 – Root Cause Analysis
Correlation analysis revealed:

- Staffing shortages contribute to longer wait times
- Higher wait times correlate with lower patient satisfaction

### Phase 5 – Staffing Optimization Simulation
Simulated a **15% wait reduction during peak hours**, showing:

- Reduced average wait time
- Fewer patients waiting over 60 minutes
